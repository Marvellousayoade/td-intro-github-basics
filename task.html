Define Version Control

Version control systems are software tools that help developers track changes made to code over time and collaborate with other developers on the same codebase


What do you understand by Git?

Git is a version control system that is used to keep track of changes made to files and folders in a project. It was created by Linus Torvalds in 2005 for the development of the Linux kernel. Git is different from other version control systems because it thinks about its data more like a stream of snapshots. This makes Git more like a mini file system with some incredibly powerful tools built on top of it


Describe the process of committing a project to github repository

  1. Create a new repository on GitHub: Sign in to GitHub and create a new empty repository. You can choose to either initialize a README or not.
  2. Initialize your project directory as a Git repository: Open your Terminal and type “git init.
  3. Add files in your local repository: Type “git add . to add all files in your local repository.
  4. Commit the files that you have added to your local repository: Type “git commit -m” “First commit”.
  5. Add the remote repository URL as origin in your terminal: Type “git add origin .
  6. Push your code to GitHub: Type “git push -u origin.
